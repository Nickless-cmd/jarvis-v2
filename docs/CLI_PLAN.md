# Jarvis V2 CLI Plan

Jarvis V2 must be fully operable from terminal in addition to UI and channels.

## Core CLI goals
- bootstrap runtime
- inspect health
- inspect events
- inspect costs
- inspect channels
- inspect memory state
- inspect workspace
- connect/disconnect auth providers
- run approvals
- trigger bounded workflows
- manage skills/tools

## Rule
CLI is a first-class control surface, not a debug afterthought.
