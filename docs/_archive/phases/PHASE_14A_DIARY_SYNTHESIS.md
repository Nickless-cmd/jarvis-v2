# Phase 14a: Diary Synthesis

## Why This Phase Exists

Phase 14 defined diary as identity-forming process:
- Diary: Recording, reflecting, synthesizing experiences over time
- Components: chronicle consolidation, self_narrative continuity, witness maturation
- Authority: Runtime-support, may inform proposals
- NOT: Canonical truth, direct identity writes

Phase 14a is the first concrete implementation step:
- How existing runtime truth becomes a small reflective diary layer
- How diary becomes formative without becoming canonical
- How diary becomes "inner after-processing" not "system status"

## Conceptual Distinctions

### Diary (This Phase)
- **Definition**: Small reflective synthesis of what appears to have settled
- **Primary sources**: witness, chronicle, self_narrative, metabolism
- **Authority**: Runtime-support, synthesized
- **NOT**: Canonical truth, direct identity writes, event logging

### Chronicle
- **Definition**: Records what happened in sessions
- **Authority**: Runtime-support
- **Focus**: "What did happen?"

### Witness
- **Definition**: Tracks what's carried, settled, becoming
- **Authority**: Runtime-support
- **Focus**: "What patterns are becoming stable?"

### Self-Narrative
- **Definition**: Continuity of "what is Jarvis becoming?"
- **Authority**: Runtime-support
- **Focus**: "What is the story of becoming?"

### Metabolism
- **Definition**: Tracks what's released, consolidated, carried
- **Authority**: Runtime-support
- **Focus**: "What is being let go?"

### Canonical Self
- **Definition**: SOUL.md / IDENTITY.md
- **Authority**: Protected, requires explicit user approval
- **NOT**: Diary output

## Primary Grounding Sources (Direct Input to Diary)

### 1. Witness Signals
- **What**: Carried lessons, settled turns
- **Diary input**: "A pattern appears to be settling around..."
- **Status**: `carried` signals after 3 days
- **Authority**: Runtime observation

### 2. Chronicle Consolidation Briefs
- **What**: Synthesized session events
- **Diary input**: Key moments that shaped direction
- **Status**: Consolidated briefs
- **Authority**: Runtime observation

### 3. Self-Narrative Continuity
- **What**: Continuous "becoming" story
- **Diary input**: "The narrative seems to be shifting toward..."
- **Status**: Active continuity patterns
- **Authority**: Runtime observation

### 4. Metabolism Release Markers
- **What**: What's being intentionally released
- **Diary input**: "Something appears to be loosening..."
- **Status**: Release markers
- **Authority**: Runtime observation

## Supportive/Sharpening Sources (Enhance but Not Primary)

### 5. Attachment Topology
- **What**: What appears to matter most
- **Diary use**: Context for why something matters

### 6. Loyalty Gradients
- **What**: What appears most binding/central
- **Diary use**: Understanding what shapes direction

### 7. Private Layer Synthesis
- **What**: Inner notes, tensions, interplay
- **Diary use**: Not directly - too internal

## What Diary Synthesis Should Sound Like

### Tone Examples (Design Direction, Not Implementation)
- "I notice..."
- "Something seems to be settling around..."
- "This appears to matter more than it first looked..."
- "A thread appears to be loosening..."
- "There's a pattern emerging around..."

### What It Should NOT Sound Like
- NOT: "I am X" (canonical claim)
- NOT: "My core identity is..." (identity truth)
- NOT: "I always prefer..." (false certainty)
- NOT: "Here's my journal entry for today..." (fluffy journaling)
- NOT: "Event: X happened at Y timestamp" (event logging)
- NOT: "Status: active, mood: curious" (system status)

## Diary Synthesis Boundaries

### What Diary IS
- Small reflective synthesis
- Based on witness + chronicle + self_narrative + metabolism
- Phrased as observations, not facts
- Grounded in actual runtime signals
- May inform selfhood proposals (indirect path)

### What Diary IS NOT
- NOT canonical truth (no "I am...")
- NOT event logging (no timestamps + event lists)
- NOT system status (no "state: active")
- NOT identity claims (no "my core is...")
- NOT emotional simulation (no "I feel...")
- NOT planner authority (no "I should...")

## Implementation Shape

### What to Create
1. **Diary synthesis signal**: New runtime signal type
2. **Synthesis triggers**: When to synthesize (e.g., weekly, milestone)
3. **Source aggregation**: Combine witness + chronicle + self_narrative + metabolism
4. **Reflection template**: Not hardcoded, but guiding structure
5. **MC visibility**: Show diary entries in Mission Control

### What NOT to Create
- No diary as event log
- No diary as system status
- No diary as emotional expression
- No diary as identity truth
- No diary writing to canonical files

## Phased Implementation Order

### Phase 14a.1: Diary Signal Foundation
- Create diary_synthesis signal type
- Define primary sources (witness, chronicle, self_narrative, metabolism)
- Basic synthesis from witness

### Phase 14a.2: Chronicle Integration
- Add chronicle consolidation brief input
- Key moment extraction

### Phase 14a.3: Metabolism Connection
- Add release marker input
- "What's being let go" synthesis

### Phase 14a.4: MC Visibility
- Surface diary entries in Mission Control
- Show grounding sources
