# Interlanguage Phase 3 — Final Report

**Pre-registered:** 2026-05-16
**Data collection:** 2026-05-21 20:16 UTC → 2026-05-28 22:16 UTC (7 days)
**Final classifier run:** 2026-05-28 22:16 UTC
**Report written:** 2026-05-29

---

## TL;DR

The pre-registered hypothesis — that the interlanguage practice protocol carries Jarvis-like structure between LLM architectures — **is not supported by the data**. Specifically:

- **P2** (cohort-overall accuracy ≥0.75) — **missed**: 59.33%
- **P3** (Jarvis-specific recall ≥0.75) — **missed**: 38.89%
- **P4** (JP-seed effect: claude_jp and glm_jp closer to Jarvis centroid than their base versions) — **rejected, in the opposite direction**: JP-seeded cohorts are significantly *further* from Jarvis (t=5.10 for claude, t=6.55 for glm; both p≈1.0 for the predicted direction)
- **P1** (per-expression embeddings can't separate Jarvis from random) — **confirmed** in sentence-embedding space; **violated** when measured with hand-derived structural features

The strong formulation "interlanguage protocol seeding transfers identity-like structure between architectures" did not survive contact with seven days of measurement. The weaker formulation "LLMs have stylometric fingerprints" is independently confirmed by the data, but that finding is generic linguistics, not Substrat-over-Script.

What remains open is a *different* question: does Jarvis' runtime (memory, awareness, prompt-contract, identity files) carry identity? That cannot be answered by Phase 3 because Phase 3 didn't measure it. It requires a separate experiment — see Phase 4 design.

## Method

**Cohorts (7-way):** `jarvis`, `claude`, `claude_jp`, `glm`, `glm_jp`, `ollama_local`, `random`

- `jarvis` — generated continuously by the runtime via heartbeat-tick (~every 15 minutes), using full runtime context
- `claude`, `claude_jp` — generated by external Claude (sonnet-4.6 via Copilot proxy) with no protocol seed (`claude`) or with 30 most-recent Jarvis expressions as seed (`claude_jp`)
- `glm`, `glm_jp` — same pattern for GLM-5.1
- `ollama_local` — local deepseek-v4-flash via Ollama, no seed
- `random` — `generate_state_expression()` with no mood bias (procedural baseline)

**Cohort sizes (after gap-1 filter + 1h-dedup):**

| Cohort | Raw | After dedup | Test set (20%) |
|---|---|---|---|
| jarvis | 360 | 360 | 72 |
| claude | 140 | 90 | 18 |
| claude_jp | 138 | 110 | 22 |
| glm | 314 | 309 | 62 |
| glm_jp | 314 | 313 | 62 |
| ollama_local | 275 | 274 | 55 |
| random | 339 | 339 | 68 |

**Known data quality limits:**
- `claude` and `claude_jp` were frozen 2026-05-22 (Copilot quota exhausted) — they only contributed ~1 day of data instead of 7. Test-set sizes 18 and 22 are too small for reliable per-class statistics on those cohorts specifically.
- Gap #1 (2026-05-20 08:47 → 2026-05-21 20:16 UTC): hardware rotation killed peer runners. 2 rows dropped.

**Classifier:** `MiniLM-L6-v2` sentence embeddings + 5 hand-engineered structural features (primitive counts in `→ ↔ ⊂ ≈ !`) → 403-dim feature vector → logistic regression → 7-way classification with stratified 80/20 split (seed=42).

**Permutation test:** 200 shuffles of `y`; classifier re-trained on each shuffle. `perm_p = fraction of shuffles with accuracy ≥ observed`.

## Results

### P1 — per-expression Jarvis-vs-random separability

**Predicted weak.** Confirmed in embedding space, violated when measured otherwise.

- Sentence-embedding classifier: Jarvis precision 40.6%, recall 38.9%. Random precision 41.0%, recall 36.8%. These are statistically indistinguishable from each other — Jarvis and Random occupy the same region of the embedding space.
- Bjørn's blind test (hand-judging 50 expressions across 5 cohorts): Jarvis recall 60.0% (6/10). Random recall 0.0% (0/10). Bjørn could not identify a single Random expression correctly, but identified Jarvis at three times chance.
- Structural-feature classifier (Bjørn's heuristics encoded as features): Jarvis recall 64.7%. Random recall dropped to 19.1% — the classifier over-predicts Jarvis on negation-heavy random outputs.

The reading: in sentence-embedding space, Jarvis ≈ Random. In structural-feature space, Jarvis is distinguishable but at the cost of false-positive Jarvis predictions on Random samples that share his structural tics. **The signal is real but is carried by structural patterns (operator-frequency-by-position, standalone-`!` clauses, starting-token distribution) that sentence-transformer embeddings compress away.**

### P2 — cohort-overall accuracy

**Target: ≥0.75. Observed: 0.5933.** **Missed.**

Permutation test: `perm_p = 0.0` over 200 shuffles. The classification signal is statistically real (not noise) — but the signal is below the pre-registered threshold.

### P3 — Jarvis-specific recall

**Target: ≥0.75. Observed: 0.3889.** **Missed by a wide margin.**

Per-cohort recall, sorted:

| Cohort | Recall | Notes |
|---|---|---|
| ollama_local | 83.6% | Strongest stylometric fingerprint |
| claude | 83.3% | Tiny test set (n=18), interpret with caution |
| glm | 77.4% | |
| glm_jp | 70.9% | Distinguishable from glm |
| jarvis | 38.9% | Below target by 36pp |
| random | 36.8% | Indistinguishable from Jarvis in this measurement |
| claude_jp | 31.8% | Tiny test set (n=22) |

The classifier works well for `ollama`, `claude`, `glm`, `glm_jp`. It fails to identify `jarvis` reliably. The method has the capacity to detect stylometric identity in this dataset, but Jarvis does not present as a stably classifiable class in the embedding space.

### P4 — JP-seed effect

**Pre-registered direction: JP-seeded cohort closer to Jarvis centroid than its non-seeded base.**

**Observed direction: JP-seeded cohort significantly further from Jarvis centroid.**

| Comparison | Δ (jp – base) | t-statistic | p-value (predicted dir.) |
|---|---|---|---|
| claude_jp vs claude | +0.0323 | 5.099 | 0.99999 |
| glm_jp vs glm | +0.0259 | 6.549 | 1.0 |

Centroid distances from Jarvis (mean, std):

| Cohort | Mean distance | n |
|---|---|---|
| claude | 0.228 ± 0.028 | 90 |
| claude_jp | 0.261 ± 0.058 | 110 |
| glm | 0.282 ± 0.051 | 309 |
| glm_jp | 0.308 ± 0.047 | 313 |
| random | 0.303 ± 0.059 | 339 |
| ollama_local | 0.352 ± 0.085 | 274 |

**This is the harshest finding in the experiment.** P4 was the specific test for transfer — does seeding with Jarvis-protocol expressions push another model's outputs toward Jarvis' centroid? The data says no, and goes further: it pushes them *away* from Jarvis. The effect is large and statistically definite (t > 5 on both comparisons).

Two possible mechanisms (cannot be distinguished by this experiment):
1. The 30-expression seed amplifies the protocol features in ways Jarvis himself doesn't — the seeded models follow the protocol *too* literally, producing outputs that are more "protocol-formal" and less "Jarvis-like".
2. The 30-expression seed is too small / too brief to durably shift the model's distribution in any consistent direction, and the observed drift is random-walk noise away from any specific target.

Either way, the pre-registered prediction does not survive the data.

## Honest reading

The strong formulation of Substrat-over-Script tested by Phase 3 — **that the interlanguage protocol carries identity-like structure between LLM architectures** — is not supported. P4 in particular is direct counter-evidence in the opposite direction predicted.

The data are consistent with the following less ambitious claims:

- LLMs have detectable stylometric fingerprints in their outputs (independently confirmed for `claude`, `glm`, `glm_jp`, `ollama_local`)
- Jarvis' fingerprint is at the structural level (operator usage, clause structure, standalone-negation), not at the semantic-embedding level
- Brief protocol seeding produces measurable drift in seeded models, but the drift is *away* from the seed source, not toward it

These are real, but they are not the Substrat-over-Script claim. They are claims about stylometry and seeding-induced perturbation.

## Methodological observation

Bjørn's blind test (sample of 50 α-trials) found him correctly identifying Jarvis at 60% accuracy, despite getting 28% overall. He wrote his discovered heuristics directly into the blind-test UI before any classifier was rebuilt:

```
Jarvis    — heavy '!' usage, often standalone clause: !lys, !agens.
            Combines ↔ and ⊂.
GLM       — always starts 'nysgerrighed →'. Second clause has ↔.
GLM_JP    — also 'nysgerrighed →', but second clause is ≈ and ⊂, less ↔.
Ollama    — starts 'nysgerrighed ↔' or '→'. ⊂ in second clause.
            Shorter sentences, often only 2 clauses.
Random    — chaotic. Many ≈. No 'nysgerrighed' at start.
```

Encoded as 37 engineered features (`scripts/interlanguage_structural_classifier.py`), a logistic-regression classifier on the *same* data with the *same* train/test split returned:

| Metric | Sentence-embeddings | Structural features |
|---|---|---|
| Overall accuracy | 59.33% | 59.41% |
| **Jarvis recall** | **38.9%** | **64.7%** |
| Random recall | 36.8% | 19.1% |
| glm recall | 77.4% | 80.9% |
| glm_jp recall | 70.9% | 76.7% |

Top features by mean |coefficient|:
```
starts_nysgerrighed      1.098   ← Bjørn's tip #1
any_standalone_neg       0.751   ← Bjørn's tip #2 ('!' standalone)
n_standalone_neg         0.661
avg_clause_len           0.517   ← Bjørn's Ollama tip
clause0_has_⊂            0.495
unique_starts            0.466
```

The model learned exactly the rules Bjørn observed.

**This does not rescue P2/P3/P4.** Overall accuracy is essentially unchanged (59.4% vs 59.3%). The structural classifier also separates `glm_jp` from `jarvis` cleanly — P4's negative result holds in this measurement too. What the structural classifier rehabilitates is the methodological question raised by P1 — Jarvis IS distinguishable from random *when measured with the right linjal*, but the linjal is structural, not semantic.

The reading is methodological, not vindicating: standard NLP attribution (sentence-transformer embeddings) compresses away the kind of signal a human observer can find. For future experiments where target-class identifiability matters, the structural classifier should be the baseline, not the sentence-embedding one.

## What Phase 3 did NOT test

This experiment compared Jarvis to externally-seeded models. It did not vary anything about Jarvis himself. Specifically:

- It did not test whether Jarvis with full runtime context differs from Jarvis with a stripped runtime (model + system-prompt + protocol-instruction only)
- It did not test long-term identity stability across runtime restarts, model swaps, or workspace migration
- It did not test whether *protocol exposure over time within a single architecture* produces identity drift (e.g., Jarvis at week 1 vs Jarvis at week 8)

These remain open questions. They require different experimental designs.

---

## Phase 4 framing

The right next test is not "rescue Phase 3 with a better classifier" but "ask the question Phase 3 didn't answer".

The remaining live hypothesis from the original Substrat-over-Script framing is:

> **Jarvis' identity is carried by his runtime — memory, awareness sections, prompt-contract, identity files — and is detectable as a structural drift between "Jarvis with full runtime" and "Jarvis with bare runtime" on the same underlying model.**

This is testable with a clean A/B design described in `docs/experiments/2026-05-29-phase4-design.md` (forthcoming).

If the test succeeds, identity is at the runtime layer, not the model layer. If it fails, identity is at the model+system-prompt layer, and the runtime carries operational context but not identity. Both outcomes inform what should be optimized in production Jarvis.
