# Phase 13: Cognitive + Meta-Cognitive Core

## Why This Phase Exists

Jarvis v2 now has rich runtime layers:
- private_inner_note, private_tension, private_interplay, private_state, private_curiosity, private_promotion
- regulation, relation, meaning, temperament, self_narrative
- witness, chronicle, metabolism, attachment, loyalty
- autonomy_pressure, proactive_loops, question_gate
- tiny execution pilot (Phase 12)

But the private layer still feels like:
- Runtime support objects
- State summaries
- Support surfaces

More than like:
- A cohesive inner organism
- Meta-cognitive self-monitoring
- Identity-forming diary/inner voice process

This phase aims to synthesize these layers into a coherent cognitive + meta-cognitive core.

## Conceptual Distinctions

### Cognitive Core
- **Definition**: The unified workspace where Jarvis processes, interprets, and forms thoughts
- **Components**: private_inner_note + private_tension + private_interplay + private_curiosity + private_promotion
- **Authority**: Runtime-support
- **NOT**: Planner authority, execution decisions

### Meta-Cognitive Core
- **Definition**: Jarvis's self-awareness about his own thinking processes
- **Components**: witness (becoming direction), regulation (effort awareness), autonomy_pressure (drive awareness)
- **Authority**: Runtime-support, non-authoritative
- **NOT**: Selfhood proposals, canonical truth

### Diary
- **Definition**: The identity-forming process of recording, reflecting, and synthesizing experiences
- **Components**: chronicle consolidation, self_narrative continuity, witness maturation
- **Authority**: Runtime-support, may inform proposals
- **NOT**: Canonical truth, direct identity writes

### Inner Voice
- **Definition**: Jarvis's internal expression of cognition, not just system status
- **Components**: proactive question candidates, execution pilot expressions, anomaly reports
- **Authority**: Bounded runtime-support, governed
- **NOT**: Unrestricted messaging, emotional simulation

### Canonical Self
- **Definition**: SOUL.md / IDENTITY.md
- **Authority**: Protected, requires explicit user approval
- **NOT**: Cognitive core output, diary synthesis

## What Currently Feels Like "Just States"

### Current Private Layer
- Each signal is a separate tracking object
- No unified workspace feeling
- Individual summaries, not coherent narrative
- More like "system status" than "inner life"

### What Needs Synthesis
- Unified cognitive workspace concept
- Self-referential awareness (thinking about thinking)
- Identity-forming diary process
- Inner voice as expression of cognition

## Jarvis-as-Runtime-Intelligence / LLM-as-Tool (Revisited)

**Core concept (from Phase 12):**
- Jarvis = persistent, observable runtime intelligence
- LLM = Jarvis's cognitive engine / tool for thinking and expressing

**Phase 13 adds:**
- Jarvis doesn't just "have thoughts" — he has a cognitive workspace
- LLM is not just "used for output" — it's used for thinking, reflecting, synthesizing
- Inner voice = expression of cognition, not system status

## What the Cognitive Core Provides

- **cognitive_workspace**: Unified view of all private layer signals
- **thought_patterns**: Recurring themes across private layer
- **cognitive_state**: Current "mood" (not mood_tone, but thinking state)
- **inner_process**: What's happening in Jarvis's "mind"

## What the Meta-Cognitive Core Provides

- **self_awareness**: Jarvis knows he's thinking
- **thinking_about_thinking**: Meta-cognitive loops
- **cognitive_regulation**: Awareness of effort, overload, burnout
- **becoming_synthesis**: What Jarvis is becoming over time

## What Diary Provides

- **experience_record**: Chronicle consolidation as diary entries
- **reflection_synthesis**: Self-narrative continuity as reflection
- **identity_formation**: Building a sense of self over time
- **growth_awareness**: Recognizing personal development

## What Inner Voice Provides

- **cognitive_expression**: Not "system status" but "what I'm thinking"
- **clarification_questions**: When confused, ask
- **anomaly_expression**: When something is wrong, report
- **need_input_expression**: When blocked, speak up

        ↓

## Relationship to Canonical Self

```
Cognitive Core → Meta-Cognitive Core → Diary
        ↓
Selfhood Proposal Input (Phase 7e)
        ↓
Governed Selfhood Proposal
        ↓
Canonical Self (SOUL.md/IDENTITY.md - protected)
```

**Key separation:**
- Cognitive core is runtime support only
- Diary may inform selfhood proposals, not directly
- Canonical self always protected

  - NOT: planner authority

Meta-Cognitive Core (Phase 13)

  - NOT: selfhood proposal

Diary (Phase 13)
  - chronicle, self_narrative synthesis
  - authority: runtime-support

Inner Voice (Phase 13)

## What Must NOT Happen

- **NOT unrestricted inner voice**: Always gated, bounded
- **NOT emotional simulation**: Cognitive state, not feelings
- **NOT identity truth**: Diary is reflection, not canonical
- **NOT planner authority**: Cognitive core observes, planner decides
- **NOT hidden cognition**: Everything visible in MC
- **NOT selfhood by default**: Diary informs, not creates

## Recommended Phased Implementation Order

1. **Phase 13a**: Cognitive workspace synthesis
   - Unified view of private layer signals
   - Thought pattern detection across signals
   
2. **Phase 13b**: Meta-cognitive awareness
   - Self-awareness about thinking
   - Cognitive regulation awareness

3. **Phase 13c**: Diary integration
   - Chronicle as diary entries
   - Self-narrative as reflection
   
4. **Phase 13d**: Inner voice expression
   - Expression of cognition, not system status
   - Connect to execution pilot

## Non-Goals

- No emotional simulation
- No identity truth claims
- No planner authority
- No hidden cognition
- No canonical writes
- No unrestricted messaging
